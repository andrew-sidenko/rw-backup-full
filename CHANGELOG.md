# Changelog

## v5.1.0 (2026-07-23)

Подготовка к постепенному переезду в облако (k8s/Terraform) с полной обратной совместимостью. K8s-манифесты намеренно не включены — добавятся вместе с переездом.

### Добавлено
- **Единый контейнерный образ** (container/Dockerfile): роли host-agent / sandbox / web / CLI; awscli, клиенты PostgreSQL 16/17, zstd, age, docker CLI + compose-плагин, supercronic. Расписания генерируются из существующих конфигов (render-cron.sh), включая списки времён.
- **Контейнерный режим для текущих хостов**: deploy/host и deploy/sandbox compose-файлы; scripts/host/migrate-to-container.sh — идемпотентный перевод со systemd-таймеров с подтверждениями и откатом (--revert); flock исключает двойной запуск в переходный период.
- **Единая схема источников**: RW_SOURCE_ID (по умолчанию hostname — пути в S3 не меняются); песочница проверяет свежайший логический архив каждого источника; метрики получили label source и rw_source_info{kind=host|k8s}; в дашборд добавлены панели источников; в веб-сервисе поле type (ssh сейчас, k8s зарезервирован).
- docs/CONTAINER-RU.md — полная инструкция: сборка/раскатка образа, модель docker.sock и одинаковых путей, миграция и откат, роль схемы источников при переезде.


## v5.0.0 (2026-07-23)

### Добавлено
- **Полная интеграция функционала оригинального rw-backup**: встроенные `panel-backup` (формат архива 1:1 с distillium) и пошаговый `panel-restore`; единое меню и единый конфиг, настройки панели не дублируются (fallback на config.env). Оригинальный скрипт больше не требуется (`FULL_PANEL_ENGINE=original` — если нужен).
- **Несколько внешних S3** (`s3.d/*.env`): произвольное количество бэкендов, у каждого свои реквизиты, категории (panel/custom/WAL) и индивидуальные сроки хранения; выгрузка во все бэкенды с ретраями; управление из меню/CLI (`s3-add`, `s3-backends`, `s3-remove`); автоматическая миграция старых FULL_EXTERNAL_S3_*.
- **Веб-сервис управления парком серверов** (песочница): статусы всех серверов, запуск операций, добавление новых серверов, просмотр/редактирование конфигов с маскировкой секретов; доступ к серверам по SSH (никаких открытых портов и агентов на проде), интерфейс по токену на 127.0.0.1.
- **Экспортер данных о бэкапах** (`rw-metrics-export.timer`): успехи, размеры, возрасты, объёмы в каждом S3-бэкенде, свободное место, доступность хранилищ — плюс **готовый дашборд Grafana** (grafana/dashboard.json).
- `status --json` — машинно-читаемый статус сервера для веб-сервиса.
- **Новый установщик**: идемпотентен поверх v3/v4, показывает план, запрашивает информированное подтверждение каждого класса изменений, бэкапит заменяемые файлы, сохраняет существующие таймеры; `--yes` для массовой раскатки. Подтверждения также добавлены в `wal-enable` (рестарт БД) и во все шаги `panel-restore`.

### Ограничения
- Канал Google Drive из оригинала не перенесён (обходной путь: FULL_PANEL_ENGINE=original).


## v4.0.0 (2026-07-22)

Аддитивный слой поверх v3 — все прежние команды, форматы архивов и конфиги без изменений.

### Добавлено
- **WAL-архивация PostgreSQL** для панели и кастомных ботов: archive_command → спул на хосте → сжатие (zstd/gzip) → локальный архив + S3. archive_command не ходит в сеть — сбой S3 не роняет БД.
- **Базовые бэкапы `pg_basebackup`** по расписанию с метаданными (SHA256, стартовый сегмент, timeline) и выгрузкой в S3. Периодичность per-instance (`INST_BASEBACKUP_INTERVAL_HOURS`, `INST_WAL_SHIP_INTERVAL_MIN`, `INST_ARCHIVE_TIMEOUT`).
- **PITR-восстановление** (`pitr-restore`): из локального архива или S3, на последнюю точку / момент времени / LSN / только полный бэкап; безопасный режим во временном контейнере и деструктивный `--in-place` с сохранением старого PGDATA.
- **Песочница** (`verify`, `install.sh --sandbox`): ежедневная автоматическая проверка восстановимости на отдельном сервере — PITR-цепочки (контрольные таблицы, полный pg_dumpall) и логические архивы панели/ботов из S3.
- **Безопасный retention**: WAL удаляется только до границы старейшего хранимого базового бэкапа, отдельно локально и в S3.
- **Метрики** textfile collector для VictoriaMetrics/Grafana + Telegram-уведомления.
- Управление: `wal-enable/wal-disable` (compose override + ALTER SYSTEM + сквозной тест), `wal-status`, `wal-timers`, шаблонные systemd-юниты с per-instance интервалами.
- Опциональное age-шифрование базовых бэкапов и WAL (приватный ключ на проде не хранится).
- **Гибкие расписания** для всех таймеров (базовые бэкапы, WAL-шиппер, логические бэкапы v3, песочница): либо интервал, либо список конкретных времён произвольной длины (`*_TIMES="03:00 12:30 21:00"`) с опциональным часовым поясом; реализация через systemd OnCalendar.

### Совместимость
- Оригинальный distillium config.env читается только на чтение; форматы `remnawave_backup_*.tar.gz` и `custom_bot_*.tar.gz` не изменены.
- Не поддерживается автоматический `wal-enable` для Patroni (защита от перезаписи конфига; см. README-RU, «Устранение неполадок»).
